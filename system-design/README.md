## 1. 로드 밸런싱

### 목적
서버가 여러 개일 때 응답이 한 곳으로 모이면 부하 발생
→ 요청을 여러 서버에 분산

### 알고리즘

**Round Robin (RR)**
- 순차적으로 서버에 분산

**IP Hashing**
- 클라이언트 IP를 해시하여 특정 서버에 매칭
- 동일 IP는 항상 같은 서버로 라우팅

**Least Connections**
- 연결 수가 가장 적은 서버에 할당

**Weighted Round Robin**
- 서버 성능에 따라 가중치 부여

## 2. 트래픽 폭주 대응

### 대응 방법

**Auto Scaling**
- 트래픽에 따라 서버 자동 확장/축소
- **Scale Up**: 수직 확장 (서버 성능 향상)
- **Scale Out**: 수평 확장 (서버 개수 증가)

**Cache**
- 자주 사용하는 데이터 캐싱
- 서버 부하 감소

**Queuing**
- 요청을 큐에 저장하여 순차 처리
- 서버 과부하 방지

**Rate Limiting / Throttling**
- 요청 수 제한
- DDoS 공격 방어

**쿼리 최적화**
- 데이터베이스 쿼리 성능 개선

### Scale Up vs Scale Out

| 항목 | Scale Up | Scale Out |
|------|----------|-----------|
| 방식 | 수직 확장 | 수평 확장 |
| 내용 | 서버 성능 향상 | 서버 개수 증가 |
| 비용 | 높음 | 상대적으로 낮음 |
| 확장성 | 제한적 | 높음 |

## 3. 웹 서버 vs 웹 애플리케이션 서버

### 웹 서버 (Web Server)

**역할**
- **정적 데이터** 처리
- HTML, CSS, JavaScript, 이미지 등

**예시**
- Nginx, Apache

### 웹 애플리케이션 서버 (WAS)

**역할**
- **동적 데이터** 처리
- 요청에 따른 비즈니스 로직 처리
- 데이터베이스 연동

**예시**
- Tomcat, JBoss

### 웹 구조

**웹 서버 - WAS 구조**
- 웹 서버가 정적 파일 처리
- WAS가 동적 요청 처리
- 로드 밸런싱 및 필터링으로 부하 감소

### 비교

| 항목 | 웹 서버 | WAS |
|------|---------|-----|
| 처리 내용 | 정적 데이터 | 동적 데이터 |
| 비즈니스 로직 | 없음 | 있음 |
| 예시 | Nginx, Apache | Tomcat, JBoss |

## 4. Proxy (Forward vs Reverse)

### Proxy란?
클라이언트와 서버 사이에서 **중간자 역할**을 하는 서버나 소프트웨어

### Forward Proxy (포워드 프록시)

**위치**
- 클라이언트 측

**역할**
- 클라이언트의 요청을 대신 전달
- 요청자 주소 보안 (서버는 프록시 IP만 확인)
- CORS 요청 에러 해결

### Reverse Proxy (리버스 프록시)

**위치**
- 서버 측

**역할**
- 서버의 응답을 대신 전달
- **로드 밸런싱**
- DDoS 공격 방어
- SSL 복호화
- 캐싱

### 비교

| 항목 | Forward Proxy | Reverse Proxy |
|------|--------------|---------------|
| 위치 | 클라이언트 측 | 서버 측 |
| 목적 | 클라이언트 보호 | 서버 보호 및 부하 분산 |
| 로드 밸런싱 | 불가 | 가능 |
